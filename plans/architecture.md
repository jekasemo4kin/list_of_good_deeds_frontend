│   │   └── register/      # Страница регистрации
│   ├── layout.tsx         # Корневой layout
│   └── page.tsx           # Главная страница приложения
├── components/            # React компоненты
│   ├── auth/              # Компоненты аутентификации
│   │   ├── Login.tsx
│   │   └── Register.tsx
│   ├── ui/                # Базовые UI компоненты
│   │   ├── SearchBar.tsx
│   │   ├── TitleBar.tsx
│   │   ├── Button.tsx
│   │   └── Input.tsx
│   ├── todos/             # Компоненты работы с делами
│   │   ├── TodoItem.tsx
│   │   ├── TodoList.tsx
│   │   └── CreateTodoModal.tsx
│   ├── friends/           # Компоненты системы друзей

## Поток данных

### 1. Аутентификация
```
Пользователь → Login/Register → API → JWT cookie → Redux store → Доступ к приложению
```

### 2. Работа с делами
```
UI → Redux actions → API → База данных → WebSocket → Обновление UI
```

### 3. Система друзей
```
UI → Redux actions → API → WebSocket → Обновление UI
```

## WebSocket события

### Широковещательные события
- `todo_updated` - создание/обновление/удаление дела
- `user_deleted` - удаление пользователя
- `friendship_updated` - изменение дружбы

## Состояние приложения (Redux Store)

### Auth slice
- `user` - текущий пользователь
- `isAuthenticated` - статус аутентификации
- `loading` - состояние загрузки

### Todos slice
- `todos` - список дел
- `searchQuery` - текущий запрос поиска
- `titleFilter` - фильтр по названию
- `nameFilter` - фильтр по автору

### Friends slice
- `friends` - список друзей
- `friendsModalOpen` - состояние модального окна

## API интеграция

### REST API
- `POST /auth/register` - регистрация
- `POST /auth/login` - вход
- `DELETE /auth/profile` - удаление аккаунта
- `GET /todos` - поиск дел
- `POST /todos` - создание дела
- `PUT /todos/:id` - редактирование дела
- `DELETE /todos/:id` - удаление дела
- `GET /friends` - список друзей
- `POST /friends/:userId` - управление дружбой

### WebSocket
- Подключение к `ws://localhost:3001`
- Обработка широковещательных событий

## UI компоненты

### Header
- Кнопка "Log Out"
- Текст "Привет, {username}!"
- Кнопка "удалить пользователя"

### SearchBar
- Инпут для title
- Инпут для name
- Кнопка "search"

### Кнопки действия
- "Все Дела" - показать все дела
- "Мои Друзья" - показать модальное окно друзей
- "Создать Дело" - показать модальное окно создания дела
- "Мои Дела" - показать только мои дела

### TitleBar
- Динамический текст о текущем состоянии

### ResultList
- Список TodoItem

### TodoItem
- Название дела (центр)
- Кнопка "show/hide" (справа)
- Кнопки действия (слева) - динамические в зависимости от пользователя:
  - **Для автора дела:**
    - "edit" → при нажатии превращается в "save" и "cancel"
    - "save" сохраняет изменения title/description
    - "cancel" отменяет редактирование
  - **Для других пользователей:**
    - "подружиться" → при нажатии превращается в "отписаться"
    - "отписаться" → при нажатии превращается в "подружиться"
- Кликабельное имя автора (при клике выполняет поиск по этому имени)

### Модальные окна
- FriendsListModal - список друзей
- CreateTodoModal - создание дела

## Адаптивность

- Mobile-first подход
- Tailwind CSS для стилизации
- Гибкая верстка для разных размеров экранов

## Отличия между layout.tsx и page.tsx

**layout.tsx** - корневой шаблон всего приложения:
- Определяет общую структуру страницы (header, main, footer)
- Управляет состоянием аутентификации (перенаправление на login если не авторизован)
- Содержит глобальные стили и мета-теги
- Оборачивает все дочерние страницы

**page.tsx** - главная страница приложения:
- Содержит конкретный контент для главной страницы
- Использует layout.tsx как родительский компонент
- В нашем случае это будет основной интерфейс приложения (SearchBar, кнопки, ResultList)

## Назначение компонентов

**Button.tsx** - базовый компонент кнопки:
- Универсальная кнопка с разными стилями (primary, secondary, danger и т.д.)
- Поддержка disabled состояния
- Адаптивный размер и отступы

**Input.tsx** - базовый компонент поля ввода:
- Универсальное текстовое поле с валидацией
- Поддержка placeholder, disabled, required
- Стили Tailwind CSS

**TodoItem.tsx** - компонент отдельного дела:
- Отображает title дела по центру
- Справа кнопка show/hide для раскрытия description
- Слева динамические кнопки (edit для автора, friend для других)
- Содержит кликабельное имя автора

**TodoList.tsx** - компонент списка дел:
- Обертка для массива TodoItem
- Обработка пагинации или бесконечной загрузки
- Состояние загрузки и ошибок

**CreateTodoModal.tsx** - модальное окно создания дела:
- Popup окно с формой (title, description input)
- Кнопки save/cancel
- Затемнение фона при открытии

**MainLayout.tsx** - основной layout приложения:
- Содержит Header с кнопками Log Out, Delete User, приветствием
- Оборачивает основной контент приложения
- Управляет состоянием модальных окон
